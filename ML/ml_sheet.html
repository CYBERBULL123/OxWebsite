Machine Learning Cheat Sheet

A futuristic guide to top algorithms and models in ML and Deep Learning.

Supervised Learning

Linear Regression

Definition: Predicts continuous values by fitting a linear relationship between input and output.

Workflow: Minimizes the sum of squared errors to find the best-fit line.

Key Concept: Assumes a linear relationship between independent and dependent variables.

Logistic Regression

Definition: Used for binary or multi-class classification by estimating probabilities.

Workflow: Applies a sigmoid function to map predictions to probabilities.

Key Concept: Uses maximum likelihood estimation to optimize parameters.

Decision Trees

Definition: Splits data into branches to make decisions based on feature values.

Workflow: Recursively partitions data to maximize information gain.

Key Concept: Prone to overfitting; often used in ensembles like Random Forest.

Random Forest

Definition: Ensemble of decision trees for high accuracy and reduced overfitting.

Workflow: Builds multiple trees and averages their predictions.

Key Concept: Uses bagging (bootstrap aggregating) to improve performance.

SVM

Definition: Finds the optimal hyperplane to separate classes in high-dimensional space.

Workflow: Maximizes the margin between classes using support vectors.

Key Concept: Can use kernel functions for non-linear classification.

k-NN

Definition: Classifies data points based on the majority class of their nearest neighbors.

Workflow: Computes distances (e.g., Euclidean) to find the k-nearest neighbors.

Key Concept: Lazy learner; no training phase, but computationally expensive during prediction.

Gradient Boosting

Definition: Builds models sequentially to correct errors from previous models.

Workflow: Fits new models to the residuals of previous models.

Key Concept: Uses gradient descent to minimize loss functions.

Naive Bayes

Definition: Probabilistic classifier based on Bayes' theorem, ideal for text data.

Workflow: Calculates posterior probabilities using prior and likelihood.

Key Concept: Assumes independence between features (naive assumption).

Neural Networks

Definition: General-purpose model for classification and regression tasks.

Workflow: Uses layers of neurons to learn complex patterns.

Key Concept: Backpropagation adjusts weights to minimize error.

Ensemble Methods

Definition: Combines multiple models to improve performance and robustness.

Workflow: Uses techniques like bagging, boosting, and stacking.

Key Concept: Reduces variance and bias by leveraging diverse models.

Unsupervised Learning

k-Means Clustering

Definition: Partitions data into k clusters based on similarity.

Workflow: Iteratively assigns data points to the nearest centroid and updates centroids.

Key Concept: Sensitive to initial centroid placement; requires predefined k.

Hierarchical Clustering

Definition: Builds a hierarchy of clusters using dendrograms.

Workflow: Agglomerative (bottom-up) or divisive (top-down) approaches.

Key Concept: No need to specify k; results in a tree-like structure.

DBSCAN

Definition: Density-based clustering for identifying outliers.

Workflow: Groups dense regions of data points and marks outliers as noise.

Key Concept: Does not require predefined k; handles noise well.

PCA

Definition: Reduces dimensionality while preserving variance in data.

Workflow: Projects data onto orthogonal axes with maximum variance.

Key Concept: Eigenvalues and eigenvectors determine principal components.

t-SNE

Definition: Visualizes high-dimensional data in 2D or 3D.

Workflow: Minimizes divergence between high-dimensional and low-dimensional distributions.

Key Concept: Focuses on preserving local relationships between data points.

GMM

Definition: Probabilistic model for clustering using Gaussian distributions.

Workflow: Fits multiple Gaussian distributions to the data.

Key Concept: Soft clustering; assigns probabilities to cluster membership.

Apriori Algorithm

Definition: Finds frequent itemsets for association rule mining.

Workflow: Uses a breadth-first search to identify frequent itemsets.

Key Concept: Based on the "apriori principle" (subsets of frequent itemsets are also frequent).

ICA

Definition: Separates mixed signals into independent components.

Workflow: Maximizes statistical independence between components.

Key Concept: Used in blind source separation and signal processing.

Autoencoders

Definition: Neural networks for dimensionality reduction and feature learning.

Workflow: Encodes input data into a lower-dimensional representation and decodes it back.

Key Concept: Can be used for anomaly detection and generative tasks.

SOM

Definition: Creates a low-dimensional representation of data for visualization.

Workflow: Maps high-dimensional data to a 2D grid using competitive learning.

Key Concept: Preserves topological properties of the input data.

Reinforcement Learning

Q-Learning

Definition: Learns optimal policies for Markov Decision Processes.

Workflow: Updates Q-values using the Bellman equation.

Key Concept: Model-free; learns directly from interactions with the environment.

DQN

Definition: Combines Q-Learning with deep neural networks for complex tasks.

Workflow: Uses experience replay and target networks for stability.

Key Concept: Handles high-dimensional state spaces (e.g., images).

Policy Gradient Methods

Definition: Optimizes policies directly using gradient ascent.

Workflow: Adjusts policy parameters to maximize expected rewards.

Key Concept: Can handle continuous action spaces.

PPO

Definition: Ensures stable and efficient policy updates.

Workflow: Clips policy updates to avoid large deviations.

Key Concept: Balances exploration and exploitation effectively.

Monte Carlo Methods

Definition: Learns from complete episodes of interaction.

Workflow: Updates value estimates based on total rewards from episodes.

Key Concept: Requires episodic tasks; high variance but unbiased.

SARSA

Definition: On-policy algorithm for learning action-value functions.

Workflow: Updates Q-values based on the current policy.

Key Concept: Learns the value of the policy being followed.

DDPG

Definition: Handles continuous action spaces using actor-critic architecture.

Workflow: Uses separate networks for policy (actor) and value function (critic).

Key Concept: Combines the benefits of policy gradients and Q-learning.

A3C

Definition: Asynchronous RL for parallel training of agents.

Workflow: Multiple agents explore the environment and update a global model.

Key Concept: Speeds up learning by leveraging parallel computation.

TRPO

Definition: Ensures stable policy updates using trust regions.

Workflow: Constrains policy updates to prevent large deviations.

Key Concept: Provides monotonic improvement in policy performance.

SAC

Definition: Maximizes both rewards and policy entropy for efficient learning.

Workflow: Uses entropy regularization to encourage exploration.

Key Concept: Balances exploration and exploitation effectively.

Deep Learning

CNN

Definition: Specialized for image recognition and processing.

Workflow: Uses convolutional layers to extract features and pooling layers for downsampling.

Key Concept: Captures spatial hierarchies in images.

RNN

Definition: Handles sequential data like time series and text.

Workflow: Processes sequences step-by-step, maintaining a hidden state.

Key Concept: Struggles with long-term dependencies; improved by LSTM/GRU.

Transformers

Definition: State-of-the-art for NLP and image tasks.

Workflow: Uses self-attention mechanisms to process input sequences in parallel.

Key Concept: Eliminates the need for recurrence; highly scalable.

GAN

Definition: Generates realistic data by pitting two networks against each other.

Workflow: Generator creates fake data; discriminator distinguishes real from fake.

Key Concept: Adversarial training leads to high-quality outputs.

Autoencoders

Definition: Used for dimensionality reduction and generative tasks.

Workflow: Encodes input data into a lower-dimensional representation and decodes it back.

Key Concept: Can be used for anomaly detection and feature learning.

U-Net

Definition: Designed for image segmentation tasks.

Workflow: Combines a contracting path (encoder) with an expansive path (decoder).

Key Concept: Skip connections preserve spatial information.

YOLO

Definition: Real-time object detection system.

Workflow: Divides the image into a grid and predicts bounding boxes and class probabilities.

Key Concept: Single forward pass for detection; highly efficient.

Siamese Networks

Definition: Used for similarity learning tasks like face recognition.

Workflow: Compares two inputs using shared weights and computes a similarity score.

Key Concept: Effective for one-shot learning.

NTM

Definition: Neural networks with external memory for complex tasks.

Workflow: Reads from and writes to an external memory matrix.

Key Concept: Mimics the working memory of the human brain.

Capsule Networks

Definition: Improves image recognition by capturing spatial hierarchies.

Workflow: Uses capsules to represent entities and their spatial relationships.

Key Concept: Addresses limitations of CNNs in capturing spatial information.
